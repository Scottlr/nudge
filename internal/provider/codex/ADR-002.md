# ADR-002: Use the Codex app-server stdio protocol

## Status

Accepted for Nudge v1.

## Decision

Nudge launches the trusted Codex executable with `codex app-server` and uses
its default stdio transport. Stdout is treated as bounded newline-delimited
JSON protocol data and stderr is retained only as bounded diagnostics. Raw
version-specific wire values remain under `internal/provider/codex/protocol`;
the application sees only provider-neutral values and events.

Nudge does not use a WebSocket listener, direct model API, or terminal-screen
scraping. The client defaults to the stable app-server surface and does not
opt into experimental API fields. The checked-in schema bundle is regenerated
from the exact supported executable version before intentional protocol
changes.

## Consequences

- A single managed child process owns the connection and serialized stdin.
- Frame size, queue count, and queue resident bytes are bounded before routing.
- Responses, notifications, and server requests are distinguished by the
  JSON-RPC-shaped envelope without adding a generic JSON-RPC dependency.
- Unknown additive notifications are safe to ignore; malformed or oversized
  frames fail the connection and pending calls.
- A Codex upgrade requires schema regeneration, fixture review, and an
  intentional adapter diff.
